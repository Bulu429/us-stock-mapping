🗂️W1 映射总结 · 两根主干（NVDA / GOOGL）
▼
英伟达和谷歌的产品线布局决定行业走势，因此具备"点石成金"的能力。映射他们有两种思路： ① BOM 拆解——拆单元节点的价值量构成；② 技术墙——在他们正攻关的物理墙上找优势企业（技术主干）。
📌 W1 产业总结：不管是英伟达还是谷歌，单计算节点的算力提升都已撞墙（功耗/散热/制程瓶颈）——节点本身做不大了，只能靠把更多节点连接起来堆算力。因此互联（节点之间 + 节点内部）的重要性正在系统性变高，这是 W2/W3 光互联、PCB 成为映射主线的根本产业逻辑。
映射标的（按"是否解决技术问题"分主次）
标的对应逻辑展开
光互联 主线 单元节点之间的互联 连接的节点越多 → 光互联需求越大 W2 / W3
PCB 主线 单元节点内部的互联 节点越大、速率 / 通道越高 → PCB 需求持续提升；无海外对标 → NVDA / GOOGL 是唯一映射源 本周

🗂️W2 映射总结 · Scale-up 互联两个核心供应商（AVGO / MRVL）
▼
NVDA 用 NVLink 定义了 Scale-up（节点内 / 节点间互联）的私有标准；AVGO 和 MRVL 是这一层的两个核心供应商—— AVGO 主导开放以太网交换芯片（Tomahawk）并为各家定制 ASIC 做配套芯片；MRVL 提供连接生态的全栈产品（光 DSP / SerDes / Retimer / CPO）。 二者的产品节奏和指引，决定了 Scale-up 从铜向光切换的速度。
📌 W2 产业总结：W1 判断"节点撞墙后转向连接"，在 Scale-up 层面的具体兑现就是铜缆向光 / CPO 的切换。 AVGO 的交换芯片路标（102.4T Tomahawk 6 量产、200G→224G SerDes）和 MRVL 的光 DSP / TIA 出货节奏， 是判断 A 股交换芯片、PCB 载板、光模块、CXL/Retimer 四条链景气拐点的最直接信号。
💡 方法论沉淀：SerDes 是 AVGO / MRVL 互联业务共同的物理"根"——博通的交换芯片（Tomahawk 512×200G SerDes）、定制 XPU 的 I/O die，以及两家的光 DSP，本质都是同一代 SerDes 工艺在不同产品形态上的封装；"互联"这条主线落到技术指标，最终就是 SerDes 代际（112G→200G→224G）+ 光电转换的迭代节奏。 但护城河不止 SerDes 本身——真正的卡位是把 SerDes 与switch fabric / FEC 算法 / 先进封装捆绑成系统级芯片的能力，这也是为什么交换芯片、光 DSP、定制 ASIC 三块业务被同一套 SerDes 底座连在一起、产品节奏高度同步。
